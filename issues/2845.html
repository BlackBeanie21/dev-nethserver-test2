Release file and MOTD

Status:CLOSEDStart date:

100%

Category:nethserver-release
Target version:v6.6-beta1
Resolution: NEEDINFO:No

Description

After every upgrade of centos-release rpm, both /etc/system-release and /etc/redhat-release files are replaced with the ones from the package.
In this case the NethServer rebrand is lost (see /etc/e-smith/events/actions/nethserver-release-rebrand).

I'd like to remove the current rebrand of both files.
Eventually we can add a MOTD pointing out some basic information about the NethServer installation, for example:


***** Welcome to NethServer *****

This is a NethServer installation. Before editing configuration files, be aware of the automatic event and templates system.
See developer manual for more information.

To enable the MOTD in SSH daemon, make sure to set PrintMotd option to yes.
