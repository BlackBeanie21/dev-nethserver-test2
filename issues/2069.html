Bug #2069

Postfix is not accessibile from red interface when server is in gateway mode

Status:CLOSEDStart date:
Priority:NormalDue date:
Assignee:-% Done:

100%

Category:nethserver-mail-server
Target version:v6.4-beta2
Security class: Resolution:
Affected version:v6.4-beta1 NEEDINFO:No

Description

Postfix is not accessible from the external interface, when the server has at least one red interface and acts as gateway: the mail server can't receive any mail from the outside world.

Expected behavior: Postfix can accept connections from internet.

Workaround:

config setprop postfix access public
signal-event firewall-adjust
