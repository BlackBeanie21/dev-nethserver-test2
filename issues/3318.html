Description

By now the UI does not validate IDN character sequences on host and domain names. We could add IDN support by fixing the host and domain validator logic.

IDN is standardized https://tools.ietf.org/html/rfc34903 with ACE prefix (xn--).
And from year 2009 allowed and now widely used for example in EU TLD http://www.mpo.cz/dokument66986.html1
Without repaired bug "add IDN domain record" in to user interface is not usable in Europe. I mean the unusable for general users.
IT technician can use a configuration files, but then litle becomes meaningless use NethServer.

Thanks to delusion for having reported this:
http://community.nethserver.org/t/idn-broken-in-ui/2128/3
