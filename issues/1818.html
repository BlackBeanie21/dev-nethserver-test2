Description

The system doesn't allow relay to external messages when a client is using the SMTP port 25.
This behavior can lead to a reconfiguration of all mail clients when migrating from NethService.

How can we address this issue?

History

#1

I think we should encourage the use of submission ports on new installations. On (authenticated) submission ports some specific policies can be enforced: disclaimers, signing, spam scoring...

Currently the port number is the only way to determine if the traffic is coming from a user/submission agent and apply the disclaimer feature, because of amavisd-new design.

For migrations, we can provide a "legacy" configuration that enables SMTP AUTH on port 25 but loses the disclaimer feature.

#2

Port 465 and 587 should behave the same.
Port 25 could accept tls auth if a db property is set for compatiblity.
