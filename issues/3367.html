Description

i would report an issue that can drive ppl to make some very annoying/dangerous mistakes

- nethserver enterprise --> page RestoreData
- instead of looking manually for the folder to be restored you search a word in the textbox
- you key a word and use "enter" for executing
- you select one or more folder or you do some click in the folder tree
- then you come back above in the page in order to search another word
- you put another word and use again "enter"

the last "enter" do not run a second search but the restore process on the last selected folder in the tree, at least this is the behaviour i noticed

in my opinion, when user put the focus in the search textbox the page code should make the "enter" key come back to run the search and not the restore, because it is against every intuitive meaning and start totally unwanted restore processes

tnx, have a nice day
