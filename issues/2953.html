Allow to keep coova-chilli's interface ip address

Status:CLOSEDStart date:
Priority:NormalDue date:
Assignee:-% Done:

100%

Category:coova-chilli
Target version:v6.5
Resolution: NEEDINFO:No

Description

Actually the coova-chilli's init script drop the ip address on the physical interface where chilli runs.
In some complex configurations it would be kept.

Suggested Implementation: add HS_LANIF_KEEPADDR option in the init script to keep the ip address on the physical interface.

Test Case

  1. Install the package on a clean machine (S) connected to Internet and to a private network.
  2. Configure chilli properly and set firewall rules.
  3. Start the service with 
    service chilli start
  4. Verify that the physical interface assignated to coova-chilli * DOES NOT HAVE * its ip address.
  5. Stop the service with 
    service chilli stop
  6. Edit (or create) the sysconfig coova-chilli file (/etc/sysconfig/chilli) and set HS_LANIF_KEEPADDR to 0.
  7. Repeat steps 3, 4, 5.
  8. Edit the sysconfig coova-chilli file (/etc/sysconfig/chilli) and set HS_LANIF_KEEPADDR to 1.
  9. Repeat step 3.
  10. Verify that the physical interface assignated to coova-chilli * HAS * its ip address.
