Bug #3428

On a _NethServer_ joined to an _AD domain_ the installation of @"nethserver-collectd"@ package adds a fragment to the file @"/etc/hosts"@ used to attribue the DNS name @DNS.domain.tld@ to the primary DNS server configured in _NethServer_.
That fragment breaks the correct reverse lookup of primary dns server name that in AD scenarios is the _AD PDC itself_.

*Before* nethserver-collectd @&quot;nethserver-collectd&quot;@ installation:

<pre># config show dns
dns=configuration
NameServers=192.168.0.110

# host dc1.domain.tld
dc1.domain.tld has address 192.168.0.110

# host 192.168.0.110
110.0.168.192.in-addr.arpa domain name pointer dc1.domain.tld.</pre>

*After* nethserver-collectd @&quot;nethserver-collectd&quot;@ installation:

<pre># config show dns
dns=configuration
NameServers=192.168.0.110

# host dc1.domain.tld
dc1.domain.tld has address 192.168.0.110

# host 192.168.0.110
110.0.168.192.in-addr.arpa domain name pointer DNS.domain.tld.</pre>

This configuration prevents communication between NethServer _NethServer_ and _AD PDC_ needed for _Kerberos_ ticket renewal.
