Feature #2740

In a multi-ISP scenario is a common need to route certain traffic using a specific provider.

The firewall should be able to manage mangle rules. rules inside the @tc@ databas. A rule can be something like this inside the @tc@ database: this:
<pre>
1=rule
Src=host;myhost Src=zone;green
Dst=0.0.0.0/0
Service=service;ssh
Provider=provider;myadsl
status=enabled
Description=
</pre>

Where:
* key: numeric id
* Src: can be a host, an zone, or a custom value like IP or CIDR
* Dst: can be a host, an zone, or a custom value like IP or CIDR
* Provider: provider name to use for this kind of traffic
* Service: (optional) can be a service object
* status: can be enabled or disabled. Default is enabled
* Description: (optional)
