Bug #2883

When accessing the server-manager on @https://<hostname>:980@ the UA is redirected to<pre>
https://<hostname>:980/<preferredlanguage>/Dashboard
</pre>
Actually, @<preferredlanguage>@ comes from HTTP client headers, and depends on user's settings. Only @en@ and @it@ languages are accepted. For other language codes the following error message is displayed:<pre>
Nethgui:

404 - Language not found [1377519247
</pre>
See attachment:language-not-found.jpg

Thanks to Petr Pe&#x0161;ava Pesava for signalling this.
